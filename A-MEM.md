# A-MEM：Agentic Memory for LLM Agents

核心思想：是让 Agent 的记忆不只是存起来再检索，而是能够主动组织、主动关联、主动演化。

## 一.研究背景

### 1.为什么需要记忆系统？

虽然 GPT、Claude 等大模型已经具备较强推理能力，但 Agent 在真实环境中需要：

- 长时间与用户交互
- 记住历史事件
- 从过去经验中学习
- 利用旧知识解决新问题

因此必须有长期记忆（Long-term Memory）。

### 2. 现有方法的问题

当前主流记忆系统（MemGPT、MemoryBank 等）主要采用：

```
存储 → 检索 → 使用
```

存在三个问题：

**（1）记忆结构是预定义的**

开发者必须提前规定：

- 什么内容存储
- 什么时候存储
- 如何组织
- 如何检索

缺乏灵活性。

**（2）知识不会成长**

例如 Agent 学到一个新的数学解题方法：

传统系统只能：记下来

但不会：发现与旧知识的联系、更新已有理解

**（3）无法适应开放环境**

现实世界知识不断变化：旧经验 ←→ 新经验

应该持续相互影响。

而传统记忆库更像：静态数据库

不是：动态知识网络

## 二.研究方法

作者受到德国学者 Niklas Luhmann 的 **Zettelkasten（卡片盒笔记法）** 启发。

其思想是：每条知识是一个独立笔记，并通过大量链接形成知识网络。

### 1.Note Construction（记忆构建）

传统做法保存：用户说的话 时间

A-MEM做法：每条记忆会被转换成一个“智能笔记”。

| 属性        | 含义   |
| --------- | ---- |
| Content   | 原始内容 |
| Timestamp | 时间   |
| Keywords  | 关键词  |
| Tags      | 标签   |
| Context   | 语义描述 |
| Embedding | 向量表示 |
| Links     | 关联记忆 |

举例：

用户说：”我的缓存系统内存占用太高“

A-MEM 不只是存这句话而会生成：

```
关键词：cache, memory, optimization
标签：software, backend
上下文：用户正在开发缓存系统，遇到了内存优化问题
```

### 2.Link Generation（记忆连接）

当新记忆进入时：

```
新记忆
 ↓
找最相似历史记忆
 ↓
建立链接
```

具体流程：

**Step1**

Embedding 检索 Top-k

利用余弦相似度：cos(new, old)

找到最相关记忆。

**Step2**

LLM 判断是否真正相关、不是简单看向量距离。

而是让 LLM 思考：

```
它们是否属于同一主题？是否存在因果关系？是否能形成知识链？
```

然后决定是否建立链接。

### 3.Memory Evolution（记忆演化）

传统系统：旧记忆永远不变

A-MEM新记忆到来时：不仅更新自己还会更新旧记忆

例子：

第一次：我在开发缓存系统

后来：缓存系统采用LRU淘汰策略

传统系统：两条独立记录

A-MEM旧记忆会被更新为（LLM判断）：

```
开发缓存系统
使用LRU策略
目标是优化内存
```

形成更完整知识。

**为什么重要？**

类似人类学习：

```
新知识
↓
重新理解旧知识
↓
形成新认知
```

### 4.Memory Retrieval（记忆检索）

查询时：

```
当前问题
↓
Embedding
↓
Top-k检索
```

找到相关记忆。

但与传统RAG不同：

由于记忆已经形成网络：

```
相关记忆+关联记忆
```

都会被带出来。

例如：

问：缓存系统怎么优化？

系统可能同时检索：

```
缓存设计
LRU策略
内存问题
```

形成更完整上下文。

### 工作流程

**Step 1：构建记忆（Note Construction）**

Agent 与用户交互后，不会直接保存原始对话。

A-MEM 会利用 **LLM**  自动生成：

```
内容(Content)关键词(Keywords)标签(Tags)上下文(Context)向量(Embedding)
```

形成一个结构化记忆笔记。

**Step 2：建立关联（Link Generation）**

新记忆加入后：

```
新记忆
↓
Embedding检索
↓
找到Top-k相似记忆
↓
LLM判断是否相关
↓
建立链接
```

系统会自动建立关联，形成知识网络。

类似知识图谱自动构建。

**Step 3：记忆演化（Memory Evolution）**

新记忆加入后：会更新旧记忆

LLM判断：

```
哪些旧记忆需要更新
更新成什么样
```

**Step 4：记忆检索（Memory Retrieval）**

当用户再次提问：

系统会：

1.将问题编码成向量

2.检索最相关记忆

3.自动扩展关联记忆

因此检索到的不只是一条记忆，而是一组相关知识。

## 三.实验设计

### 实验设置

**数据集**

作者使用两个长上下文数据集。

**1.LoCoMo**

专门评估长期对话记忆。

特点：

- 平均9000 token
- 最长35轮会话
- 7512个问答

包含：

- Single-hop
- Multi-hop
- Temporal
- Open-domain
- Adversarial

五类任务。

**2.DialSim**

长时间多人对话数据集。

来源：

- Friends
- Big Bang Theory（生活大爆炸）
- The Office

约35万Token。

**对比方法**

比较：

- LoCoMo
- ReadAgent
- MemoryBank
- MemGPT
- A-MEM

**评测指标**

| 指标               | 含义    |
| ---------------- | ----- |
| F1               | 答案准确率 |

Ollama（部署Qwen和Llama）

## 四.总结

A-MEM = 向量记忆库（Vector Memory） + 记忆图（Memory Graph） + LLM驱动的记忆重写（Memory Rewriting）机制。

### 创新点

1.让新记忆主动修改旧记忆，而不是简单追加。

2.把 Memory 写成 Graph

### 不足

1.Memory Evolution 可能产生幻觉。LLM可能脑补历史信息

2.没有版本管理，新记忆覆盖旧记忆

### Q&A

**1.A-MEM 和 MemGPT 的区别是什么？**

MemGPT 的核心是上下文管理，将记忆分为主存和外存，通过调度机制解决上下文窗口有限的问题。A-MEM 关注的是记忆组织方式。它不仅存储记忆，还会自动生成语义描述、自动建立记忆连接、自动更新旧记忆。

因此 MemGPT 更像操作系统中的内存管理，而 A-MEM 更像知识组织与知识更新机制。

**2.为什么要引入 Link Generation？**

如果只依赖向量检索，每条记忆都是孤立的。A-MEM 希望形成关联结构。这样检索到 A 时，还能自动扩展到 B 和 C，从而为复杂推理任务提供更多上下文信息，特别适用于 Multi-Hop Reasoning。

**3.这个方法最大的风险是什么？**

最大的风险是 **Memory Drift（记忆漂移）**。因为旧记忆更新完全由 LLM 生成。

如果某次更新出现幻觉：错误会不断传播和放大。论文没有设计校验机制，也没有分析长期运行后的漂移问题，这是比较大的缺陷。

**4.它跟RAG有什么区别**

RAG 的知识库是静态的，只负责“检索”；A-MEM 的记忆库是动态的，不仅能够检索，还能通过 Link Generation 建立记忆关系，并通过 Memory Evolution 用新经验持续更新旧记忆，因此它更像一个会成长的记忆系统，而不是传统意义上的检索增强系统。

**5.这篇文章是否真正解决了长期记忆问题？**

没有完全解决。它改善了记忆组织与利用方式，但仍然依赖：

- Embedding召回
- LLM生成
- 人工设计Prompt

并没有解决：

- 记忆真实性
- 记忆遗忘机制
- 记忆压缩
- 百万级长期学习

等更根本的问题。

**6.你认为这篇论文最大的不足是什么？**

我认为最大的不足是：作者证明了 A-MEM 在长对话问答上有效，但没有证明它能够显著提升真实 Agent 的任务执行能力。

实验主要基于 LoCoMo 和 DialSim 等 QA 数据集。

如果能够在：

- Web Agent
- Coding Agent
- Tool Agent

等真实环境中验证长期学习效果，结论会更有说服力。
