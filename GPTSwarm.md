# GPTSwarm: Language Agents as Optimizable Graphs

**ICML 2024 oral**

**核心思想**：将多智能体 LLM 系统统一表示为**可优化图（Graph）**，通过节点（Node）和边（Edge）优化，实现自动化的多 Agent 协作系统。

## 一.研究背景

随着LLM（如GPT-4、GPT-3.5）在自动化任务和问题解决上的广泛应用，人们提出了各种Prompt工程和多智能体协作框架。

问题：

- 各种 Agent 框架彼此割裂；
- 大量 prompt 和工作流靠人工手写；
- Agent 之间怎么协作、怎么通信，缺乏统一抽象；
- 很难自动优化。

受心智社会（Society of Mind）启发，论文提出将智能体系统建模为计算图。

- **节点（Node）**：执行具体操作，如LLM推理、工具调用等。
- **边（Edge）**：表示节点之间信息流和协作模式。
- **图（Graph）**：节点组合形成单个智能体。
- **群体图（Swarm）**：多个智能体图组成的层级系统。

---

## 二.研究方法

### 1. 三层结构

- **Node**：基本操作单元，包括 LLM 推理、工具调用、函数调用、行动等
  
  例如：Python 执行、Google Search、文件解析
- **Graph（Agent）**：由多个 Node 组成的 DAG（有向无环图），表示单个 Agent 的内部逻辑和工作流。
  
  例如：ReAct Agent：Question → Reasoning → Tool → Answer
- **Composite Graph（Swarm）**：由多个 Agent Graph 组成的组合图，通过 Edge 定义 Agent 间通信，实现多 Agent 协作。

图结构优势：可以统一表示单 Agent、Multi-Agent、Tree-of-Thought、Reflexion 等方法。

图执行：DAG 

- 每个节点接收输入 x 和前驱输出 z
- 输出传递给后继节点

---

### 2.优化（Optimization）

论文认为既然 Agent 是图，那图结构本身也可以被优化。

**1. Edge Optimization（边优化）**

即：优化 Agent 之间怎么通信。

多 Agent 系统里：谁跟谁通信？哪些 Agent 应该连接？

通常都是人工写死。论文认为这是可以自动学习的。

方法：

每条边都有一个概率参数，表示这条边存在的概率。

于是整个图变成：一个概率图分布。

优化目标：最大化任务效果。

例如：准确率、reward、success rate

优化算法：

REINFORCE 策略梯度：随机采样图 → 执行任务 → 得到 reward → 更新概率参数。

**2.Node Optimization（节点优化）**

传统 Prompt Optimization：只优化一个全局 Prompt。

但 Agent 系统：每个节点都有 prompt。

方法：

每个 Node维护：输入历史、输出历史、performance

然后：不断用 LLM 自己改 Prompt。

特点：局部优化，保证节点职责分离，不干扰其他节点

---

### 工作流程

- **执行阶段**：群体图接收输入，节点按拓扑顺序执行，输出答案。
- **评估阶段**：根据任务的效用函数（accuracy、正确率等）评估当前执行结果。
- **优化阶段**：
  - **节点优化**：更新各节点的提示（prompt）
  - **边优化**：调整智能体间的通信边
- **下一轮执行**：用优化后的节点/边再次执行任务，收集新的性能数据。
- **重复迭代**：不断循环，直到性能收敛或达到预定指标。

## 三.实验设计

### 1 MMLU（多任务语言理解）

**对抗性实验（Adversarial Setting）**：部分智能体故意输出错误答案

**优化目标**：通过边优化剔除有害智能体，提高群体性能

**评估方法**：多数投票、基线对比

**结果**：优化后的群体性能接近或超过单智能体基线

### 2 Mini Crosswords

**任务类型**：开放式文字填字游戏

**实验设计**：三类智能体 TOT、Reflexion、COT 组成群体

**优化方法**：边优化，通过 REINFORCE 迭代调整智能体间通信

**评估指标**：单题准确率（正确填字数量）

**结果**：优化后群体性能显著优于初始随机连接或传统方法

### 3 HumanEval（Python 编程题）

**优化方法**：节点优化，改进节点提示，加入输入输出示例

**结果**：优化后准确率从 0.76 提升到 0.88（Page 7, Figure 5）

### 4 GAIA（通用 AI 助手任务）

**任务类型**：多模态问题，需要推理、网页搜索、文件分析等工具

**评估方法**：构建同类智能体群体，使用提示自一致性（prompt-based self-consistency）

**结果**：GPTSwarm 显著优于 GPT-3.5、GPT-4 及 AutoGPT

---

## 四.总结

### 创新点

把多智能体系统抽象为可优化的图，节点和边都能自动改进，实现自动化、多任务适用的协作智能体框架。

### 不足

**计算成本较高**：边优化和节点优化需要大量 LLM 推理；对复杂任务或大规模群体，训练/优化时间和资源消耗大。

**依赖 LLM 能力**：节点优化和边优化效果受基础 LLM影响，会直接限制群体性能。

## Q&A

**1.节点优化细节**

节点执行过程中会收集 **输入-输出历史**，也就是节点在过去执行 LLM 时的上下文和结果。

利用这些历史数据，再结合节点的功能描述，GPTSwarm 使用提示优化方法（如 OPRO）调用 LLM 来生成改进后的提示词。

新的提示词会替换旧提示，再用于下一轮节点执行，从而提高节点输出的质量和正确性。

**2.边优化细节**
GPTSwarm 把每条潜在边当作一个概率：这个概率越高，边越可能存在。然后使用 REINFORCE 算法迭代优化这些概率：每一轮，框架会随机生成一组边连接，执行任务并评估群体表现，再根据结果调整边的概率，让表现好的连接被保留，表现差的减少。经过多轮迭代，边的概率会收敛到最优值，从而自动形成最有效的智能体通信结构。
