# MasRouter: Learning to Route LLMs for Multi-Agent System

**ACL 2025**

---

## 一、背景

多智能体系统（MAS）通过多个 LLM 协作提升复杂任务能力，但存在：

- **成本高**：多 agent → token / API 消耗大  
- **模型选择难**：大模型不总是最优，小模型更便宜但有时足够好  

现有方法不足：

- 单 agent 路由：只选模型  
- 固定 MAS：结构不可变  
- 缺少统一决策：**结构 / 角色 / 模型**

👉 目标：**按任务动态选择最优 MAS 配置**

---

## 二、方法

提出**级联式路由框架**：

> 协作结构 → 角色分配 → LLM 选择

### 1. 协作结构（Collaboration Determiner）

选择 agent 交互方式：

- Chain / Tree / Debate / Graph  
- 简单 → Chain；复杂 → Debate  

特点：根据问题语义预测结构

---

### 2. 角色分配（Role Allocator）

将任务拆为子任务并分配角色：

- 如：架构师 → 程序员 → 测试  

特点：**级联生成（cascade）**——后续角色依赖前序角色

---

### 3. LLM 选择（LLM Router）

为每个角色选模型：

- 数学 → 数学强模型  
- 编程 → 代码模型  

依赖：任务类型 + 角色 + 协作结构

---

### 4. 优化目标

在**性能 vs 成本**之间权衡：

- ↑ 准确率 / pass@1  
- ↓ token / API 成本  

整体流程：

> 输入问题 → 难度判断 → agent 数量 → 协作结构 → 角色 → LLM → 执行

---

## 补充

- 决策是离散的 → 用Policy Gradient（强化学习）——类似试错学习
- 数据：benchmark（题目 + 标准答案）

训练循环：

> 采样配置 → 运行 MAS → 得分（正确 + 成本）→ 更新策略  

本质：**学习式搜索（非暴力枚举）**

输出：一个**智能调度器（Router）**

**本质上是一个学习模型**：

MasRouter 的三个核心模块（协作模式决策、角色分配、LLM 路由）都是可训练的控制网络。它通过**性能和成本反馈**不断优化。

训练是让系统学会规则和分配策略，学好了之后，用户输入就能直接得到答案。

（知识 + 推理 + 编程）

### 指标

### 结果

- **性能提升**：平均 +3.5%，MBPP +8.2%  
- **成本降低**：
  - HumanEval ↓52%  
  - MBPP ↓40%+  
- **可插拔**：可集成到现有 MAS 框架中，降低 17%~28% 的开销

## 四、总结

### 创新

**提出了一个新问题**：首次清晰定义并形式化了“多智能体系统路由”这个问题，把过去分开考虑的“合作方式、角色分工、模型选择”放到了同一个框架里来解决。

### 不足

- **训练成本高**（需反复运行 MAS）  
- 依赖标准答案（RL reward）  
- 系统复杂度较高

# Q&A

**1. MasRouter 是如何工作的？**

MasRouter 包含三个模块：协作模式决策器、角色分配器和 LLM 路由器。它根据查询定制多智能体系统，先选择协作模式，再分配各智能体角色，最后为每个角色选择最适合的 LLM。

**2.论文主要解决了什么问题？**

在一个多智能体系统中，如何为每个查询智能地分配协作模式、角色以及合适的 LLM，从而在性能与成本之间取得平衡。

**3.作者如何验证 MasRouter 的有效性？**

作者在五个基准数据集上对比了 20 个基线方法，包括单智能体、多智能体及单 LLM 路由方法，评估指标涵盖性能（accuracy/pass@1）和成本（token、API 调用等）。

**4.为什么单智能体路由方法不能直接用于多智能体系统？**

单智能体路由只需为单个LLM做选择，而MAS路由还需额外解决三个问题：协作模式的选择——不同任务适合不同的通信拓扑（链式、树状、全连接图等）；智能体数量的动态确定——根据难度调整专家数量；角色分配——为智能体分配合适的专业角色。

**5.方法的可扩展性和泛化能力如何？**

当新的LLM（如DeepSeek-V3）加入候选池时，无需重新训练，模型会自动选择新模型，性能随之从84.25%提升至85.40%。此外，集成到现有MAS框架时也无需改造原框架，具有较强的通用性。
