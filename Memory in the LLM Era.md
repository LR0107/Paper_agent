# Memory in the LLM Era: Modular Architectures and Strategies in a Unified Framework

**VLDB 2024**

## 一.研究背景

近年来，像 GPT-5、Qwen3、Claude Sonnet 等大型语言模型推动了人工智能的发展，并广泛应用于工业自动化、个人助手、软件工程任务等场景。

智能体需要能够：

- 自主规划与执行多步任务
- 长期保持上下文信息
- 支持个性化和跨会话交互

**记忆的重要性**
LLM的上下文窗口有限，如果没有记忆模块：

- 早期对话信息容易丢失
- 长期推理和多轮任务性能下降

记忆机制允许智能体：积累历史知识、支持迭代推理、自我演化与个性化行为

**现有问题**

- 记忆方法繁多，但缺乏统一框架和系统比较
- 多数研究关注整体性能，而缺少对模块化组件的深入分析

---

## 二.研究方法

作者提出了一个**统一的模块化框架**，将智能体记忆方法分为四个核心模块：

### 1.信息提取（Information Extraction）

从对话或任务信息中抽取关键知识

方法包括：

- **直接归档（Direct Archive）**：存储原始消息，最简单。
- **摘要提取（Summarization-Based）**：用LLM生成信息摘要，减少冗余信息，便于后续管理。
- **图结构提取（Graph-Based）**：将对话信息解析为实体-关系-对象三元组，构建知识图，支持动态更新和时间推理

---

### 2.记忆管理（Memory Management）

目标：维护、整合和优化记忆状态。

核心操作：

**关联（Connecting）**

- 将语义相似或时间接近的记忆条目建立联系。
- 方法：图结构边、语义链接。

**整合（Integrating）**

- 合并碎片化信息，生成高层抽象表示。
- 方法：摘要、主题总结。

**跨层转换（Transforming Across Memory Levels）**

将短期记忆迁移至中期，再到长期存储。

方法：基于访问频率、最近使用时间或热度评分。

**更新（Updating）**

保持记忆准确性和一致性。

方法：

- **规则驱动**：如MemoryBank使用遗忘曲线。
- **LLM驱动**：生成摘要、合并或冲突解决。
- **智能体驱动**：自动决定修正/合并/裁剪（MemGPT、MemOS）。

**过滤（Filtering）**

移除过期或低价值记忆。

方法：

- 使用频率或时间衰减
- 基于内容的重复检测或LLM判断

---

### **3.记忆存储（Memory Storage）**

目标：高效保存处理后的记忆，支持快速检索。

- **平面存储（Flat）**：统一空间，简单但不易分层管理。
- **分层存储（Hierarchical）**：短期、中期、长期三层，结合不同管理策略。
  - 例如：MemoryOS短期存对话，中期存主题摘要，长期存用户偏好。
- **向量存储（Vector-Based）**：将文本编码为高维向量，用ANN检索。
- **图存储（Graph-Based）**：保留结构信息，多跳推理能力。
  - 例如：MemTree使用层次树，Zep使用时间分层知识图。

---

### **4.信息检索（Information Retrieval）**

目标：在生成回答时，检索最相关记忆。

方法包括：

- **向量检索（Vector-Based）**：利用语义嵌入捕捉查询与记忆间语义相似性
- **结构化检索（Structure-Based）**：利用树/图结构的多跳路径获取相关信息
- **词汇检索（Lexical-Based）**：基于关键字匹配，保证精确检索
- **LLM辅助检索（LLM-Assisted）**：将模糊查询解析为精准检索指令，发现隐含语义

## 三.实验设计

比较10种代表性记忆方法（A-MEM、MemGPT、Mem0、Mem0g、MemTree、MemOS、MemoryOS等）。

**数据集**

- **LOCOMO**：十个长对话任务，适合多轮问答评估
- **LONGMEMEVAL**：500个问题，测试信息提取、多会话推理、知识更新、时间推理

**评估指标**

- **F1**：衡量答案与参考答案的匹配程度
- **BLEU-1**：反映生成答案的词汇精确度

**硬件**：8块NVIDIA A100 80GB GPU

**检索设置**：Top-k = 10, 最大上下文长度20k tokens

**实验结果：**

- **结构优势**：树形/分层结构可以保留高层抽象信息和底层细节。
- **信息组织**：关联操作（connecting）对多跳和跨会话任务显著提升性能。
- **成本与效率**：高性能往往伴随高Token消耗，MemoryOS在保证精度的同时降低成本。
- **上下文与位置敏感**：记忆越靠近查询，越容易被利用。
- **LLM依赖**：大模型（如GPT-4o-mini）性能显著提升，开源模型性能受限，但结合结构优化可缓解。

## 附录：提出一种新的记忆结构

树形结构 + 分层存储 + 分阶段更新 + 多模式检索



**树形结构**

将记忆以树状结构组织，节点表示记忆条目，树的层级表示信息的抽象级别。

**上层节点**：保存高层概念或摘要信息

**下层叶节点**：保留细粒度对话内容、事实或原始消息

支持**多跳、多步推理**，适合长对话、多轮问答。

**分层存储**

将记忆按时间和功能划分为**短期、中期、长期**三层。

**工作方式**：

1. **短期存储**：保存最近对话和交互，高访问频率，更新频繁
2. **中期存储**：整合短期信息生成主题或事件摘要，减少冗余
3. **长期存储**：保存用户偏好和跨会话的稳定事实，更新最慢

**分阶段更新**

智能体的记忆分为**短期、中期、长期三层**。

分阶段更新意味着**不同层级的记忆以不同方式、不同时间被更新或迁移**，而不是把所有信息直接覆盖或统一更新。

流程：

- 短期记忆接收最新对话
- 中期记忆整合短期重要信息
- 长期记忆保存稳定信息

## 四.总结

### 创新点

**统一模块化框架**：首次将LLM智能体记忆方法分解为 **信息提取、记忆管理、记忆存储、信息检索** 四个模块。

**新型高效记忆方法**：树形结构 + 分层存储 + 分阶段更新 + 多模式检索

### 不足

**系统复杂性**：组合策略实现复杂。

**实验场景有限**：数据集主要是 **对话和问答类长任务**。

对大模型（GPT-4o-mini、Qwen2.5-72B）的推理能力高度依赖，小模型或开源模型性能仍有限。
