# RouterDC: Query-Based Router by Dual Contrastive Learning for Assembling Large Language Models

**NeurIPS 2024**

**核心思想**：不同的大模型擅长不同任务，因此不应该所有问题都交给同一个模型，而应该动态选择最合适的模型。

## 一.研究背景

不同 LLM 各有所长，不存在一个模型在所有任务上都最强。

能不能把多个 LLM 组合起来？

最简单的方法叫： Ensemble（集成）

即：把同一个问题同时发给多个模型、再投票决定答案

但问题：太贵、太慢 。因为每个问题都要调用所有模型。

Routing 的思路是：

> 先训练一个“小路由器”，让它判断：“这个问题应该交给哪个 LLM？”

于是：推理时只调用一个模型、成本大幅降低、速度更快。

已有方法的问题：

论文重点批评了之前最强方法：**ZOOTER**

它的方法是：

1. 给每个模型打分
2. 把分数 softmax 归一化
3. 学习一个概率分布

问题在于：多个模型都表现不错时，监督信号会变差

softmax 后：

- 高分模型之间差异被压缩
- 分布变得“平均”
- 路由器不知道到底该选谁

论文统计发现：64% 的样本中，前两名模型分数差距非常小。

因此：只学习概率分布是不合理的。

---

## 二.研究方法

**Dual Contrastive Learning（双对比学习）**

### （1）Sample-LLM Contrastive Loss

**核心思想：靠近好模型，远离差模型**

对于一个问题：选表现最好的几个模型作为正样本；表现最差的几个模型作为负样本

然后训练：

- 问题 embedding 要接近好模型 embedding
- 远离差模型 embedding

举例：“解一个数学应用题”

RouterDC 会学到：

- 靠近数学模型 embedding
- 远离代码模型 embedding

最终：这个问题在 embedding 空间里，会自然落到数学模型区域。

旧方法：学概率分布

新方法：学语义距离

### （2）Sample-Sample Contrastive Loss

作者发现：仅靠第一种 loss，训练不稳定。

原因：相似问题可能被映射到不同区域。

比如：两道购物计算题，本质很像、embedding 却很远

因此加入： 相似问题彼此靠近

方法：

1. 先对训练问题聚类
2. 同一类问题：embedding 拉近
3. 不同类问题：embedding 拉远

没有这个loss 时：

- 训练曲线波动明显
- embedding 混乱

加入后稳定很多。

### 模型结构

RouterDC 非常轻量。结构只有两部分：

**1. Encoder**

使用一个小模型：只有 86M 参数

作用：把问题编码成向量。

**2.每个 LLM 一个 embedding**

给每个大模型分配一个“向量坐标”，用来表示这个模型擅长什么

**训练阶段**

1. 准备多个 LLM ：数学、代码、中文等不同专长模型
2. 每个问题都让所有 LLM 作答并打分——用标准答案自动计算正确率，不是LLM/人工
3. 选出：
   - 表现好的模型（正样本）
   - 表现差的模型（负样本）
4. 用 Encoder 把问题编码成 embedding
5. 用对比学习训练：
   - 问题 embedding 靠近好模型 embedding
   - 远离差模型 embedding
6. 再让“相似问题”的 embedding 更接近，提高稳定性

**推理过程**

输入问题：“求解代数方程”

步骤：

1. Encoder 生成 query embedding
2. 与各个 LLM embedding 算 cosine similarity
3. 选择最相似的模型
4. 调用该模型回答

## 三.实验设计

**In-Distribution**

| 方法       | Avg       |
| -------- | --------- |
| 最强单模型    | 54.56     |
| Voting   | 54.37     |
| ZOOTER   | 53.97     |
| RouterDC | **58.54** |

**OOD 泛化**

RouterDC：45.85%

最佳单模型：43.95%

**推理速度更快**

Voting：每次调用全部模型

RouterDC：只调用一个模型

因此：比 Voting 快约 6 倍。

## 四.总结

### 创新点

以前的方法（如 ZOOTER）：输入问题 → 预测哪个模型最好

属于： 概率分布学习——强迫只有一个“正确模型”

现实里，一个问题可能多个模型都擅长。

这时候：谁第一”其实不重要。真正重要的是：哪些模型适合这个问题

于是论文提出：不学分类，改学距离——用对比学习替代分类学习

### 不足

**1. 还是“硬路由”**：最终只选一个模型，不能真正多模型协作。

**2. 训练成本高**：训练时每个问题都要跑所有 LLM

## Q&A

**1. 为什么不用一个更强的大模型，而要做 Routing？**

因为不同 LLM 擅长不同任务，单一模型很难在所有任务都最优。RouterDC 的目标是动态选择最合适的模型，从而利用多个模型的互补能力，提高整体性能。

**2. RouterDC 和传统 Ensemble 有什么区别？**

传统 Ensemble 会让所有模型同时回答，再投票得到结果，因此推理成本很高。RouterDC 只会选择一个最合适的模型进行推理，所以速度更快、成本更低。论文中提到它比 Voting 快约 6 倍。

**3. 为什么还要加入 sample-sample loss？**

只做 query-LLM 对比时，训练不稳定，相似问题可能会被映射到不同区域。作者加入 query-query 对比学习，让相似问题 embedding 更接近，从而提高稳定性和泛化能力。

**4.RouterDC 为什么能提高 OOD 泛化？**

因为它学习的是 embedding 空间中的语义关系，而不是固定任务标签。即使是没见过的新任务，只要语义上接近某类问题，就可能被路由到合适模型，因此 OOD 表现更好。

**5.RouterDC 和 MoE（Mixture of Experts）有什么关系？**

两者都属于“专家选择”思想。区别在于：

- MoE 通常在模型内部选择专家；
- RouterDC 是在多个独立 LLM 之间做选择。

可以把 RouterDC 看成一种“外部 MoE”或“多模型调度器”。
